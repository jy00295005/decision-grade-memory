# Prompt 03: Method Mapping

Use this prompt when the conceptual design is ready and needs to be translated into a Section 3 writing blueprint.

```text
Translate the Scenario Memory Layer design into a paper-facing Section 3 / Method plan.

Important framing:
- this should support one paper, not a general platform description
- keep the claim narrow and defensible
- do not foreground the underlying implementation project
- do not imply completed implementation or empirical validation

Create:
- `manuscript/framework/method_section_mapping.md`

Include:
1. Core methodological claim
2. What is not claimed
3. Method assumptions
4. Section 3 thesis paragraph
5. A clean Section 3 outline
6. One-sentence purpose for each subsection
7. Safe claims to make
8. Claims that should remain hypotheses or future directions
9. One concise method summary paragraph
10. One concise system overview paragraph

Target method claim to preserve:
We propose a scenario memory layer that organizes distributed graph-based SME context into explicit decision episodes for decision-grade context construction.

Requirements:
- optimize for paper writing, not design notes
- tie the method back to the literature review
- make subsection responsibilities clear
- prevent overclaiming by explicitly listing non-claims
```
