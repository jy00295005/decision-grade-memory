# Project Setup MVP Prompt

## Purpose

Initialize a minimal, maintainable academic paper repository for Codex-centered research and writing.

## Prompt

```md
Please make careful, conservative repository edits. Optimize for long-term maintainability and Codex collaboration, not for showing off.

Set up a clean MVP repository for an academic paper project.

Project title:
Bridging Data, Signals, and Growth: A Graph-Based AI for SME Decision-Making

Primary objective:
Create the best minimal repository for Codex-centered academic writing in the early stage of a paper project.

Important context:
- This is an early-stage research writing repo.
- Use Markdown, not LaTeX, for the first MVP.
- The repo should be optimized for iterative drafting, literature synthesis, section-by-section writing, and later migration to TeX if needed.
- Keep everything minimal, clean, and practical.
- Do not over-engineer.
- Do not create scripts, CI, Docker, tests, datasets, or tooling unless explicitly requested.
- Do not invent citations, references, results, experiments, or claims.
- Do not add fake academic content.
- Prefer concise placeholders and TODO markers.

Tasks:
1. Create the paper-repo structure.
2. Create README.md and AGENTS.md for Codex collaboration.
3. Create docs/, manuscript/, references/, data/, experiments/, results/, prompts/, and codex/ scaffolding.
4. Create section templates, bibliography placeholder, prompt template, and session handoff.
5. Keep the repository professional, minimal, and useful for iterative paper development.
```

## Notes

- This is the repository bootstrap prompt.
- Use only for initializing or re-initializing a research repo structure.
